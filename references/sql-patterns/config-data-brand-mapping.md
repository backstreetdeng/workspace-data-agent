## 已知差异（避免误匹配）

| 短名 | 误匹配风险 |
|-----|----------|
| `比亚迪` | config_data 只有"比亚迪"，sales_import 有"比亚迪汽车工业有限公司"+"比亚迪汽车有限公司"两个全名 |
| `AITO 问界` | 注意大写 AITO + 空格，问界关联到赛力斯全名 |
| `特斯拉中国` vs `特斯拉（进口）` | config_data 是进口/国产两个分开字段，sales_import 是按公司主体 |
| `一汽-大众` vs `一汽-大众捷达` | 合资子品牌要单独映射 |
| `smart` | 仅一个短名，但对应多个销售主体 |

## 测试验证脚本

```python
import psycopg2
conn = psycopg2.connect(host='192.168.3.146', port=5432, database='vectordb', user='vectordb', password='vectordb123')
cur = conn.cursor()
# 验证 config_data 短名"比亚迪"是否能命中
cur.execute("SELECT COUNT(*) FROM config_data WHERE 厂商 = %s", ('比亚迪',))
print('比亚迪:', cur.fetchone()[0])
cur.execute("SELECT COUNT(*) FROM sales_import WHERE 企业名称 LIKE %s", ('%比亚迪%',))
print('比亚迪全名:', cur.fetchone()[0])
```

## 维护规则

1. **每季度验证**：从 `config_data_brands_raw.txt` 与 `sales_import` 全名比对，发现新出现厂商就更新映射表
2. **跨表联合查询前必看**：config_data 与 sales_import JOIN 时，**不能**直接 `t1.厂商 = t2.企业名称`（会 0 行），必须先做品牌映射
3. **映射缺失**：如果新品牌不在表中，先 `SELECT DISTINCT` 全名再人工补充映射
